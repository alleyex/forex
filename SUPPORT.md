# Support

This repository is maintained as an engineering project with documented workflows, release automation, and contribution policies.

## Before Opening An Issue

Check the existing project documentation first:

- [README.md](README.md)
- [CONTRIBUTING.md](CONTRIBUTING.md)
- [docs/development.md](docs/development.md)
- [docs/operations.md](docs/operations.md)
- [docs/versioning.md](docs/versioning.md)

If the problem is reproducible and not already covered there, open a GitHub issue with the closest matching template.

## Support Routing

- Bug reports: use the bug report issue template
- Product or workflow improvements: use the feature request template
- Security disclosures: follow [SECURITY.md](SECURITY.md) instead of opening a public issue

## What To Include

For the fastest triage, include:

- operating system
- Python version
- relevant branch or commit hash
- steps to reproduce
- sanitized logs or screenshots
- whether the problem affects training, simulation, live trading, or release tooling

## Scope

This repository does not provide anonymous real-time operational support. Public issues should focus on reproducible defects, workflow improvements, and repository maintenance.
